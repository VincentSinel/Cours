Interface - Paramètres du projet

Il est possible de définir des paramètres généraux à notre projet. Pour cela, dans l'onglet Projet, il est possible d'ouvrir les paramètres du projet.

Paramètres généraux :

On arrive sur la page ci-dessus. Nous sommes dans l'onglet Général que nous allons détailler maintenant. Le premier sous-menu ouvert est Configuration. Dans ce sous-menu, on peut changer le titre de notre jeu, sa description, sa version ainsi que son icône. Nous n'allons pas détailler tous les sous-menus mais uniquement ceux permettant de gérer des options particulièrement importantes.

Dans ce second sous-menu, Fenêtre, il est possible de modifier la taille d'affichage par défaut de la fenêtre de jeu, sa manière de se lancer (fenêtré ou plein écran).
Ce menu propose aussi un paramètre important permettant de gérer ce qui se produit lorsque l'on change la taille de la fenêtre (en particulier le passage du mode fenêtré à plein écran). C'est l'ensemble des paramètres sous le titre Étirement.

  • Le Mode définit ce qui se passe lorsque l'écran change de taille.
    • disabled signifie que la vue n'est pas changée, un pixel en jeu est un pixel dans la fenêtre quelque soit sa taille. Lorsque l'on change la taille de la fenêtre, celle-ci peut alors ne plus respecter les dimensions définies au départ. Concrètement, un écran plus grand permet de voir une plus grande partie de l'espace de jeu, il y a donc une grande différence possible d'un ordinateur à l'autre.
    • canvas_item signifie que le contenu visible dans la surface de la caméra est agrandi pour correspondre à la taille de l'écran. Le rendu a lieu après l'agrandissement. Cela respecte donc les dimensions définies pour la caméra.
    • viewport signifie que le contenu visible dans la surface de la caméra est agrandi pour correspondre à la taille de l'écran. La différence avec le mode précédent est l'ordre entre rendu et agrandissement. Le rendu a lieu avant l'agrandissement, ce qui est en général bien meilleur pour les jeux au graphisme en pixel. Cela respecte donc les dimensions définies pour la caméra.
    Configuration de base
    Mode = disabled
    Mode = canvas_item
    Mode = viewport
  • L' Aspect définit la manière dont le contenu est modifié.
    • ignore signifie que l'agrandissement pourra déformer le rendu pour s'adapter à l'écran.
    • keep signifie que le ratio de taille de l'écran reste valable et la vue est agrandie pour apparaître le plus grand possible dans la zone disponible.
    • keep_width signifie la vue est agrandie pour couvrir la largeur mais s'il reste de la marge en hauteur, une plus grande partie du jeu est alors visible.
    • keep_height signifie la vue est agrandie pour couvrir la hauteur mais s'il reste de la marge en largeur, une plus grande partie du jeu est alors visible.
    • expand signifie que la vue est agrandie pour couvrir la place maximale en respectant le ratio de l'écran. Tout l'espace restant est alors une partie du jeu visible supplémentaire.
    Configuration de base
    Aspect = ignore
    Aspect = keep
    Aspect = keep_width
    Aspect = keep_height
    Aspect = expand

Dans ce troisième sous-menu, Textures, il est possible de modifier le filtre appliqué par défaut aux textures lorsqu'elles sont affichées avec une taille différente de celle à l'origine. Pour des graphismes en pixel, il est préférable d'utiliser Nearest qui donne un aspect net aux images.

Dans ce quatrième sous-menu, Environnement, il est possible de modifier la couleur par défaut du fond de la partie de jeu.

Paramètres des touches :

Dans l'onglet Contrôles, il est possible de définir des actions liées à des touches de clavier, de manettes de jeu ou de souris. Celle-ci sera alors facilement utilisables plus tard dans les programmes de nos objets.

De nombreuses actions sont déjà intégrées pour la gestion de l'interface utilisateur, mais il est préférable de définir ses propres actions pour gérer les touches en jeu. Pour ajouter une nouvelle action, il faut lui donner un nom puis cliquer sur Ajouter.

Cela crée l'action dans le menu en dessous, il faut ensuite lui assigner des touches. Pour cela, il faut appuyer sur le bouton en bout de ligne. Il est possible d'assigner plusieurs touches à une même action.

La fenêtre suivante s'ouvre alors, la case désignée par la flèche est automatiquement sélectionnée et se remplit automatiquement lorsqu'une touche ou une combinaison de touches est pressée. Il n'y a alors plus qu'à valider.
